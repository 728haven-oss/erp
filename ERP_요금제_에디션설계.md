# ERP 요금제·에디션 설계 (사용자 수 × 옵션 단계별)

**작성일:** 2026-07-04 · **목적:** 사용 인원 수와 사용 옵션(모듈) 정도에 따라 과금·사이즈를 나누는 설계 — 타 ERP 과금 모델 벤치마킹 기반 (지시: "다른 erp 참고해서 각각 나눠줘")

## 1. 타 ERP 과금 모델 조사

| ERP | 과금 방식 | 실제 가격 | 특징 |
|---|---|---|---|
| **이카운트** | 월정액 · **사용자 무제한** · 전모듈 포함 | 가입비 20만원(1회) + 월 4만원 | 인원·모듈 구분 없음. 단순함이 무기, 대신 커스터마이징 없음 |
| **히트판21c** (귀사 사용) | **영구 라이선스 · 동시사용자 수 단위** | 싱글 385,000원 / 5유저 770,000원 (VAT포함, 1회) | 구버전 관행: 유저 수로만 구분, 기능은 판형(표준형/특수형)으로 구분 |
| **Odoo** | **사용자당 월과금 + 플랜 2단계** | Standard 약 $24.9/인·월, Custom 약 $37.4/인·월 (연납, 국가별 상이) + 호스팅 별도 | 전 앱 포함하되 플랜으로 커스터마이징 권한 차등. 사용자당 과금의 대표형 |
| **영림원(SystemEver)** | 구독(사용자·모듈 조합) + 구축비 | 견적제 | 업종 패키지·모듈 단위 협의 |
| **NetSuite/더존** | 기본료 + 사용자당 + 모듈 애드온 | 견적제(고가) | 3층 구조(베이스+시트+모듈)의 대표형 |

**패턴 3유형:** ① 무제한 정액(이카운트) ② 사용자당 과금(Odoo) ③ 베이스+사용자+모듈 애드온 3층(NetSuite류). 히트판은 구식 영구제.

## 2. 우리 ERP 에디션 설계 (권장안)

소기업(≤5인) 출발 + 협력사 포털 확장이라는 우리 구조에 맞춰 **③ 3층 구조를 단순화한 하이브리드**를 권장:
**베이스 요금(에디션) + 사용자 구간 + 선택 애드온(실비 연동 옵션)**

### 에디션 (옵션 정도에 따른 구분)

| 에디션 | 포함 모듈 | 대상 |
|---|---|---|
| **Lite** | 영업(견적·수주·납품·계산서 내부기록) + 구매(발주·입고) + 재고 기본 + 장부 조회 | 1~2인, 전표 위주 |
| **Standard** | Lite 전부 + 결재 게이트 + 메일함(개인 이메일) + 재고 전체(수불대장·적정/사장재고) + 회계 전체(일월계표·어음·청구서·부가세자료) + 검사성적서·재고실사 | 3~5인 제조사 표준 (현재 v4 기본형) |
| **Pro** | Standard 전부 + PRJ 관리(호기·공수·플랜vs실적) + 공급사 포털(RFQ·DC협상·역발행 승인) + KPI + 히트판 이관 | 협력사 협업까지 쓰는 업체 |
| **Enterprise(주문형)** | Pro 전부 + Flask 서버 구축(자체 DB·백업 서버화) + 커스터마이징 + BOM/LRP 등 v5 후보 | 전용 서버·맞춤 개발 |

### 사용자 수 구간 (사이즈)

| 구간 | 인원 | 과금 방식 |
|---|---|---|
| S | 1~2인 | 베이스 요금만 |
| M | 3~5인 | 베이스 + 인당 요금 ×(인원−2) |
| L | 6~15인 | 인당 요금 단가 10% 할인 구간 |
| XL | 16인~ | 협의(서버 사양 상향 포함) |

- 히트판식 "동시접속" 대신 **계정 수** 기준(요즘 SaaS 표준, 관리 단순).
- 포털 접속 **공급사 계정은 무과금**(Odoo의 external user 무과금 관행) — 협업 확산에 유리.
- 읽기전용(조회) 계정은 0.5인 계산 옵션.

### 예시 가격표 (월 구독, VAT별도 — 시장가 참고 기준)

| | Lite | Standard | Pro |
|---|---|---|---|
| 베이스(2인 포함) | 30,000원 | 50,000원 | 80,000원 |
| 추가 사용자(인/월) | 10,000원 | 12,000원 | 15,000원 |
| 예: 5인 사용 | 60,000원 | 86,000원 | 125,000원 |

근거: 이카운트(전기능 4만·무제한)보다 낮게 시작하고, Odoo(인당 3.5만원 수준)보다 소인원 총액이 싸도록 설계. 영구제(히트판)는 업데이트·서버 시대에 부적합해 제외하되, **온프레미스 1회 구축(Enterprise)** 은 견적제로 별도.

### 애드온 (실비 연동 항목 — 기존 "비용 경계표"와 일치)

| 애드온 | 과금 | 비고 |
|---|---|---|
| 전자세금계산서 국세청 전송(정발행·역발행) | 건당 110~200원 (중계사 실비 전가) | 홈택스 연동 카드에서 건수·비용 자동 집계(이미 v4 구현) |
| 알림톡/SMS | 건당 10~30원 | 발신프로필 가입 필요 |
| 은행·카드 자동수집 | 월 1~2만원 | 스크래핑/오픈뱅킹 실비 |
| OCR/LLM 기능 | 월 정액 or 건당 | 유료 API 실비 |
| 스토리지 추가(도면·첨부) | 10GB당 월 5,000원 | 서버형에서만 |

## 3. 시스템 반영 설계 (구현 시)

- `settings.plan = {edition:'standard', users:5, addons:{hometax:true, alimtalk:false, ...}}`
- **기능 게이트**: 각 라우트/버튼에 에디션 요구 레벨 태그 → 미달 시 잠금 아이콘+업그레이드 안내(강제 차단이 아닌 안내형).
- **사용량 미터**: 사용자 수, 월 전송 건수, 스토리지 사용량을 설정>요금 카드에 표시 — 홈택스 비용 집계(v4 기구현)와 같은 패턴으로 확장.
- 테스트(HTML) 단계에서는 과금 없음 — 플랜 전환 시뮬레이터만 두고, 실과금은 Flask+PG(포트원 등, 가입+수수료) 단계에서. **자리만 준비, 비용 명기** 원칙 유지.

## 4. 결정 필요 사항 (검토 요청)

1. 판매용(외부 공급)인지 사내+협력사용인지 — 사내용이면 과금은 "원가 배부표"로만 쓰고 게이트는 끔.
2. 예시 가격표 수준 승인 여부 (시장가 대비 저가 진입 전략).
3. 공급사 포털 무과금 원칙 동의 여부.
