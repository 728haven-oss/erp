# 히트판21c 분석 및 v4 설계서

**작성일:** 2026-07-03 · **분석 대상:** 히트판21c V4.0 for Lan (SMC판) — 귀사 사용 중 실물
**조사 방법(3원):** ① 설치본 실행 화면(메뉴바·매입 메뉴 트리 직접 확인) ② **실제 마감 백업(2026.06.15) MDB 3개 파일의 테이블 구조 해독** ③ 제조사(공영정보) 공식 기능명세(hitpan.co.kr 표준형 페이지)

> v4 정의(사용자 지시): "바탕화면에 설치된 히트판을 분석해서 견적·매입·판매·재고·회계 메뉴를 참고해 우리 ERP에 삽입. 메뉴 배치는 요즘 UX/UI에 맞게. 세금계산서 역발행·영수·청구 같은 디테일 분석. 히트판 백업 파일을 받아 우리 ERP로 인수인계가 문제없이 진행되어야 함."

## 1. 히트판21c 메뉴 구조 (확인 결과)

**상단 메뉴바(실물):** 설정(T) · 견적(Q) · 매입(I) · 판매(S) · 재고(V) · 회계(A) · A/S(E) · 기타(O) · 자료관리(D) · 자료백업(B) · 창(W) · 도움말(H) · 네트워크(N) · ▷한국SMC(주)◁

### 1-1. 매입(I) — 실물 메뉴 트리 (직접 확인)
- 매입관리 (매입 전표 입력)
- 발주관리 ▶ 발주예상량산출 / 발주서작성 / 기간별발주현황 / 품목별발주현황 / 업체별발주현황 / **납기별발주현황** / **미입고현황** / **발주단가현황** / 발주서헤더설정
- 견적의뢰서작성 ▶ (구매 RFQ)
- 각종매입현황 ▶ / 매입통계분석 ▶
- SMC자동입고분관리 (SMC 대리점 전용 — 본사 발주 자동입고)

## 2. 우리 ERP(v3)와 대응 — 이미 있는 것 / v4로 넣을 것

### 이미 v3에 있음 (중복 구현 안 함)
견적→수주→납품→계산서 흐름, 발주→입고→미입고 추적, RFQ(견적의뢰), 수정계산서 6종, 재고실사·조정, 업체별/시기별 단가 + 추천 납품처, 최근 매입가 참조, 수주 진행·미납품(잔량), 월마감 잠금·감사로그, A/S 이력(호기별), 어음·카드 결제수단(수금 회차), KPI, 카드내역 CSV.

### v4 신규 (히트판에서 가져올 것) — 8개 영역
| # | 기능 | 히트판 근거 | 우리 설계 |
|---|---|---|---|
| 1 | **메뉴 재배치(현대 UX)** | 견적/매입/판매/재고/회계 업무축 | 사이드바를 업무축 6그룹으로 재편: 홈 · 영업(견적·수주·납품·계산서) · 구매(RFQ·발주·입고) · **재고** · **회계/장부** · 협업(메일·결재·포털·PRJ·달력). 상단이 아닌 좌측 유지(모바일 대응), 그룹 접기/펼치기 |
| 2 | **세금계산서 디테일** | TX_GU, 매입계산서, 합계발행, 부가세신고자료 | ① **영수/청구 구분** 필드(영수=현금 수령 완료, 청구=외상 청구) ② **역발행(매입자발행)**: 매입 계산서를 우리가 작성해 공급사에 발행 요청 → 공급사 포털에서 승인 시 확정(전자 역발행 프로세스 시뮬레이션) ③ **합계발행**: 기간 내 여러 납품을 1장으로 합산 발행(월합계) ④ **부가세 신고자료**: 기간별 매출/매입 공급가·세액 집계표 + 인쇄 |
| 3 | **상품수불대장** | 상품수불부, 수불대장 자동정리 | 품목 선택 → 기간 내 입고/출고/조정 전 건 시간순 + 누계 잔고 열. 재고 메뉴에 배치 |
| 4 | **적정재고·사장재고** | 적정재고 대비 현재고, 재고보유월수 | 품목에 적정재고(min) 세팅 → 부족 리스트(발주 제안 연계). **사장재고**: 최근 N개월(기본 6) 출고 없는 품목 + 재고보유월수(현재고÷월평균출고) |
| 5 | **회계 메뉴 승격 + 일(월)계표** | 회계(A) 메뉴, 일계표·월계표 | 장부를 "회계" 그룹으로 승격, **일계표/월계표**(일·월별 매출/매입/수금/지급/현금·은행 잔고 요약표) 추가 |
| 6 | **어음 관리대장** | EU_* (발행/받을어음, 만기) | 수금·지급의 어음 정보를 모아 **받을어음/지급어음 대장**(만기 임박 D-7 경고, 결제/부도 처리). 대시보드 만기 카드 |
| 7 | **대금청구서** | 대금청구서 | 거래처+기간 → 전월이월·당월발생·입금·잔액 요약 + 명세 리스트 인쇄 양식(월말 청구 관행) |
| 8 | **히트판 인수인계(이관)** | 백업 MDB 구조 해독 완료 | 설정에 **[히트판 이관]** 화면: 거래처/품목/단가 **CSV·엑셀 붙여넣기 임포트**(히트판 엑셀자료변환 출력 → 필드 자동 매핑: buy_name→상호, buy_taxno→사업자번호, S_PUM+S_KU→품명·규격, S_PDAN→판매가, S_IDAN→매입가, S_JEK→기초재고, UR_*→업체별 단가). 실서비스(Flask) 이행 시 **MDB 직접 변환 스크립트**(mdbtools/UCanAccess, 위 매핑표 §1-3 기준) — 전표(IJ/TX/IU)까지 이관 |

**추가 소소 반영:** 견적서 복사 버튼(견적관리), 판매 시 업체별 최근 판매단가 자동 표시(현재 매입가 참조와 대칭), 신규 거래처/품목 전표 중 자동 등록 유도.

### v4에서 제외(자리만/문서만)
팩스송신(시대 변화 — 메일로 대체), 명함·음악·달력 등 부가기능(불필요), SMC자동입고분관리(SMC 특화 — 우리는 공급사 포털의 명세서 확정→입고 승인 루프가 동일 역할), 세트조립(BOM은 v4 후보 잔존), 창고 다중 관리(단일 창고 가정 유지, Flask 때 확장).

## 3. 세금계산서 역발행·영수/청구 상세 로직 (지시 반영)

**영수/청구 (TX_GU 대응):**
- 청구: 외상 — 공급가 수금 전 발행(월말 합계 발행 관행). 미수금 잔액에 반영.
- 영수: 현금 수령 완료 후 발행 — 발행 즉시 수금 완료 처리와 연결.
- 구현: 계산서 발행 모달에 영수/청구 라디오(기본 청구), 목록·인쇄양식에 표기. 영수 선택 시 해당 회차 수금완료 여부 확인 경고.

**역발행(매입자발행 세금계산서):**
- 실무: 구매자가 계산서 내용을 작성해 공급자에게 발행 요청 → 공급자가 승인(전자서명) → 공급자 명의로 발행됨. 국세청 매입자발행 제도/전자 역발행 관행.
- 구현: 입고/발주 화면에서 [역발행 요청] → 문서 생성(상태: 역발행요청) → **공급사 포털에 승인 카드** → 공급사 승인 시 매입 계산서 확정(SENDDT 기록), 반려 시 사유 회신. 우리 원가 확정 원칙(계산서 발행 후 불변)과 연동.
- 명기: 실제 국세청 전송은 홈택스 중계사(팝빌 등) **가입+건당 과금** 필요 — 자리만, 내부 기록으로 시뮬레이션.

**합계발행:** 거래처+기간 선택 → 미발행 납품(거래명세서) 합산 → 1장 발행, 원본 납품들에 계산서번호 역링크(IJ_TAXNO 패턴).

## 4. 인수인계(마이그레이션) 설계

1. **테스트(HTML) 단계:** 히트판 [자료관리>엑셀자료변환(표준)]으로 거래처/상품/단가를 엑셀 출력 → 우리 ERP 설정>[히트판 이관]에 붙여넣기 → 미리보기(매핑 확인) → 반영. 중복은 사업자번호/품명+규격 기준 병합.
2. **실서비스(Flask) 단계:** 마감 백업 ZIP(PYOJUN/PANDATA/POTHER.mdb)을 그대로 업로드 → 서버에서 mdbtools(UCanAccess 대안)로 §1-3 매핑표에 따라 자동 변환 — 기준정보+전표 5년치+계산서 이력+어음까지 전량 이관. **이번 백업 파일로 구조 검증 완료**(테이블·필드 전수 확인)이므로 이관 리스크 낮음.
3. 이관 검증 리포트: 건수 대사(거래처 n, 품목 n, 전표 n, 계산서 n), 금액 합계 대사(연도별 매출/매입), 불일치 목록.

## 5. 비용 경계 (신규분)
- 전자세금계산서 정발행/역발행 국세청 전송: 중계사 가입+건당 과금 (기존 명기 동일) — v4는 내부 시뮬레이션.
- 히트판 MDB 변환: 무료(오픈소스 mdbtools) — 서버만 있으면 됨.

## 6. 검증 계획
기존 자동 테스트 575건 유지 + v4 신규: 영수/청구 저장·표기, 역발행 요청→포털 승인→매입계산서 확정→원가 반영, 합계발행 합산·역링크, 수불대장 누계, 적정재고 부족 검출, 사장재고 월수 계산, 일/월계표 합계 일치, 어음 만기 D-7 검출, 청구서 이월/발생/입금/잔액 산식, 히트판 CSV 임포트 매핑·중복 병합.
